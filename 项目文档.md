# 医疗设备监控上位机系统（Medical Device Monitoring Host System）

## 项目简介

本项目是一个**基于 WinForm 的模拟医疗设备上位机软件系统**，使用 **.NET Framework 4.8** 开发，旨在高度还原真实医疗器械（如多参数监护仪、输液泵、注射泵等）的数据采集、实时监控、报警处理及医院系统集成流程。

项目严格参考 **IEC 62304** 医疗器械软件生命周期标准的设计思想，融入风险管理、软件验证（V&V）、审计日志等合规要素，适合作为应届生求职医疗器械/医疗信息化岗位的核心作品集项目。

**业务场景**：
在医院 ICU、手术室、普通病房等场景中，医疗设备通过串口或 Modbus 协议实时输出生理参数。上位机软件需承担以下核心职责：
- 稳定采集并解析设备数据
- 实时波形与数值显示（符合医护操作习惯）
- 智能报警检测与分级响应
- 数据持久化（本地存储，单机部署）
- 操作审计与权限控制

该系统强调**高实时性、高可靠性、高安全性**，充分体现医疗软件的行业特性。

## 架构定位：单机版上位机

当前阶段采用 **单机版架构**，所有模块部署在同一台工控机/工作站上。排除分布式中间件依赖，降低部署复杂度，聚焦核心医疗监控业务。

## 核心业务功能

### 1. 设备通信与数据采集模块

#### 1.1 通信方式
- **SerialPort（串口）**：模拟真实医疗设备的 RS232 通信
- **Modbus RTU/TCP**：部分医疗设备采用 Modbus 协议输出数据

#### 1.2 模拟设备数据帧协议定义

定义两种协议格式，先实现文本协议快速验证链路，后续可升级为二进制协议。

**文本协议格式（第一阶段）**：
```
HR:75,SpO2:98,NIBP:120/80,RESP:16,TEMP:36.8,TS:2026-06-07 10:30:00
```
- 参数名和值以冒号分隔，参数之间以逗号分隔
- 以换行符作为帧结束标志
- 血压格式：收缩压/舒张压
- TS：设备端时间戳

**二进制协议格式（第二阶段升级）**：
```
[0xAA][0x55][Length][DeviceID][DataBlock...][CRC16][0x0D][0x0A]
```
- 帧头：0xAA 0x55
- Length：后续数据长度（不含头尾）
- DeviceID：设备标识
- DataBlock：按预定义顺序编码的生理参数
- CRC16：Modbus CRC16 校验
- 帧尾：0x0D 0x0A

#### 1.3 设备模拟器
- 开发阶段内置设备模拟器（DeviceSimulatorService）
- 按真实监护仪数据刷新频率（1~2秒/次）生成模拟生理参数
- 支持模拟异常场景：信号丢失、参数超限、设备离线
- 真实串口桥接模式（可选）：通过虚拟串口工具连接真实设备

#### 1.4 数据解析
- 实时解析参数：心率（HR）、血氧（SpO2）、无创血压（NIBP）、呼吸率（RESP）、体温（TEMP）
- 数据合理性校验（参数范围检测、变化速率检测）
- 数据异常标记（无效值、校验失败、超时数据）
- 参数单位标准化（如体温支持 ℃/℉ 转换）

### 2. 实时监控界面（WinForm）
- 主界面采用 TabControl + SplitContainer 分屏布局
- 实时波形显示区（支持多通道叠加，使用 OxyPlot 控件）
- 参数数值大字体显示 + 趋势图
- 患者信息面板（姓名、床号、年龄、诊断、入院时间等）
- 报警灯 + 声音提示（System.Media.SoundPlayer）
- 界面主题：浅色医疗风格（蓝色/绿色主调，符合医院UI规范）
- **动态绑定**：所有 UI 参数通过 MVVM 模式绑定到 ViewModel

### 3. 报警与风险管理模块
- 阈值可配置的生理报警与技术报警
- 生理参数报警阈值参考临床指南（AHA/中国ICU质控标准）设定默认值：

| 参数 | 报警低限 | 报警高限 | 单位 |
|------|---------|---------|------|
| HR（心率） | 40 | 120 | bpm |
| SpO2（血氧饱和度） | 90 | 100 | % |
| NIBP_SYS（收缩压） | 90 | 180 | mmHg |
| NIBP_DIA（舒张压） | 60 | 120 | mmHg |
| RESP（呼吸率） | 8 | 30 | 次/分 |
| TEMP（体温） | 36.0 | 38.5 | ℃ |

- 报警分级：高危（红色）、中危（橙色）、低危（黄色）
- 报警确认（Acknowledge）、单次抑制（Snooze）、延时处理
- 报警事件记录与导出（CSV/Excel）
- 参考 ISO 14971 风险管理思想：危害识别 -> 风险估计 -> 风险控制 -> 风险评价

### 4. 数据存储模块
- **本地数据库 MySQL 8.0**：患者信息表、生理参数历史表、报警事件表、操作日志表
- 实时数据缓存：内存缓存（MemoryCache）用于当前患者的最新参数
- 历史趋势数据：按设定频率（5秒/30秒/5分钟）持久化到 MySQL
- 数据导出：支持 CSV/Excel 导出历史趋势和报警记录

### 5. 系统安全与合规模块
- 登录验证 + 角色权限（护士/医生/管理员）
- 全操作日志审计（Serilog 输出到文件 + 数据库）
- 敏感字段加密（患者姓名、身份证号等）
- 全局异常捕获与软件运行日志

### 6. 后期可扩展模块（本阶段不做）
- Redis 实时缓存层：用于多床位中央监护站的实时数据共享
- RabbitMQ 异步消息：报警信息推送至护士站大屏/手机端
- ASP.NET Web API：对外提供患者数据查询、报警推送 REST 接口
- 多床位中央监护站：集中监控多个床位的实时数据

## 技术栈

| 层面 | 技术选型 | 说明 |
|------|---------|------|
| **框架** | WinForm (.NET Framework 4.8) | 医疗行业存量系统主流选型，成熟稳定 |
| **架构模式** | 分层架构（UI + ViewModel + Service + DAL + Common）+ MVVM | 关注点分离，易于测试和维护 |
| **MVVM 框架** | CommunityToolkit.Mvvm 8.x | 微软官方维护，替代已停更的 MvvmLightLibs |
| **通信** | SerialPort + NModbus | 串口通信 + Modbus RTU/TCP |
| **数据库** | MySQL 8.0 (MySql.Data.EntityFramework 8.x) | 本地数据持久化 |
| **图表控件** | OxyPlot.WindowsForms 2.x | 医疗波形实时显示 |
| **UI 样式** | MaterialSkin.2（免费） 或 DevExpress WinForms（付费） | |
| **日志** | Serilog（文件输出 + 数据库输出） | 审计日志 + 运行日志 |
| **DI 容器** | Microsoft.Extensions.DependencyInjection | 需引入兼容 .NET Framework 4.8 的版本 |
| **工具库** | Newtonsoft.Json、AutoMapper、FluentValidation | 序列化、对象映射、输入验证 |

## 临床参数参考标准

报警阈值默认值参考以下临床指南：
- **AHA（American Heart Association）**：成人心率、血压正常范围
- **中国 ICU 患者监测指南**：血氧饱和度、呼吸率、体温监测标准
- **ISO 80601-2-61**：脉搏血氧仪设备基本安全和基本性能专用要求

> **注意**：本系统为模拟项目，报警阈值仅作演示用途。实际医疗系统必须经过临床验证和法规审批。
